Project Overview

A specialized computer vision web application designed to analyze squash performance through video analysis. The app helps players understand their technique, identify areas for improvement, and track performance metrics over time. This project combines my passion for squash with my technical skills in computer vision and data analysis.

Key Features & Implementation

  • Video Processing Pipeline: Built a comprehensive video processing system using OpenCV and MediaPipe that analyzes squash match footage frame-by-frame. The system extracts pose keypoints from each frame, tracking player movement throughout the entire video. This allows for detailed analysis of technique, positioning, and movement patterns that are crucial in squash.
  • Pose Keypoint Tracking: Implemented robust pose estimation that tracks all major body joints throughout the video. The system maintains tracking even during fast movements, occlusions, and rapid direction changes that are common in squash. This provides a complete picture of player movement and body positioning during rallies.
  • Motion Analysis: Developed algorithms to analyze movement patterns, including swing mechanics, footwork, and court positioning. The system identifies key movement phases such as preparation, execution, and follow-through for different shot types. This analysis helps players understand the biomechanics of their technique.
  • Impact Detection: Created specialized algorithms to detect racket-ball impact moments with high precision. This involves analyzing keypoint trajectories, velocity changes, and spatial relationships between the player's body, racket, and ball. Accurate impact detection is crucial for analyzing shot timing and technique.
  • Rule-Based Scoring System: Designed a sophisticated scoring system that evaluates technique based on biomechanical pose features. The system analyzes factors such as racket angle, body rotation, weight transfer, and follow-through. The scoring incorporates tolerance for natural human variation, recognizing that perfect form may vary slightly between players while still identifying significant technical errors.
  • Feature Extraction: Used NumPy and Pandas to extract and process structured features from pose data. This includes calculating angles, velocities, accelerations, and spatial relationships that are relevant to squash performance. The feature extraction pipeline transforms raw keypoint data into meaningful performance metrics.
  • Data Export & Analysis: Implemented CSV export functionality that allows users to export structured performance data for further analysis. The exported data includes frame-by-frame keypoint positions, calculated metrics, and performance scores. This enables users to perform their own analysis or track performance trends over time.

Technical Challenges

Processing video frame-by-frame while maintaining accuracy required careful optimization. The system needed to handle videos of varying lengths and qualities, and maintain pose tracking accuracy even during fast movements. The rule-based scoring system needed to balance strictness with tolerance for natural variation, requiring extensive testing and refinement.

Campus Path Finder

April – May 2025

Desktop Application

Project Overview

A desktop application that helps students and visitors navigate the University of Wisconsin-Madison campus by finding the shortest path between any two locations. The app provides an intuitive interface for exploring campus and planning routes, making it easier to navigate the large university campus.

Key Features & Implementation

  • Graph-Based Navigation: Implemented a comprehensive graph data structure representing the campus with 50+ locations as nodes and pathways as edges. Each location includes metadata such as building names, coordinates, and accessibility information. The graph accurately represents the physical layout of the campus, including walking paths, shortcuts, and building connections.
  • Dijkstra's Algorithm: Implemented Dijkstra's shortest path algorithm to find optimal routes between any two campus locations. The algorithm efficiently computes paths with sub-second response times, even when searching across the entire campus graph. This ensures a smooth, responsive user experience.
  • Modular UI Architecture: Designed a clean, modular user interface using JavaFX with three main components:
    • GraphCanvas: A visual representation of the campus map where users can see locations, paths, and the calculated route highlighted. The canvas supports zooming, panning, and interactive selection of locations.
    • ControlPanel: User controls for selecting start and end locations, triggering path calculations, and adjusting display options. The panel provides dropdown menus, search functionality, and clear visual feedback.
    • ResultsPanel: Displays the calculated route with step-by-step directions, total distance, estimated walking time, and a list of landmarks along the path. The panel updates dynamically as users select different routes.
  • Usability & Future Expansion: The modular design makes it easy to add new features such as multiple route options, accessibility preferences, real-time updates, or integration with campus bus schedules. The separation of concerns allows for independent development and testing of each component.
  • Performance Optimization: Optimized the graph representation and algorithm implementation to ensure fast pathfinding. Used efficient data structures and caching strategies to minimize computation time, resulting in consistently fast response times regardless of the selected locations.

Technical Challenges

Creating an accurate graph representation of the campus required careful mapping of physical locations and pathways. The UI needed to be intuitive while displaying complex graph data in a user-friendly way. Balancing visual clarity with information density was a key design challenge.
